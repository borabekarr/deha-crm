# src/routes/

File-based route definitions powered by **TanStack Router** (v1).

## Why TanStack Router

| Concern | Rationale |
|---|---|
| Type safety | Every route, param, and search param is fully typed end-to-end — no `useParams` casting, no string guessing. |
| File-based routing | Routes are co-located with their files; the Vite plugin auto-generates `src/routeTree.gen.ts` on every build. |
| Search-param parsing | TanStack Router has first-class, type-safe search params — critical for Deha CRM's filtered pipeline views (stage, assignee, date-range). |
| Devtools | `TanStackRouterDevtools` ships in-box; loaded only in `import.meta.env.DEV`. |
| React Router comparison | React Router v6/v7 lacks the type-safe search-param API and code-gen. TanStack Router was chosen for CRM-grade filtering requirements. |

## File conventions

| File / pattern | Purpose |
|---|---|
| `__root.tsx` | Root layout — `<header>`, `<Outlet />`, `<footer>`, devtools |
| `index.tsx` | `/` — login/landing redirect placeholder |
| `_auth/route.tsx` | Pathless layout route — wraps all authenticated routes |
| `_auth/dashboard.tsx` | `/dashboard` — sales dashboard (protected) |
| `routeTree.gen.ts` | Auto-generated by `@tanstack/router-plugin/vite` — do not hand-edit |

## Adding new routes

1. Create `src/routes/<path>.tsx` (or `src/routes/_auth/<path>.tsx` for protected routes).
2. Export `const Route = createFileRoute('<full-path>')({ component: ... })`.
3. `pnpm build` (or `pnpm dev`) regenerates `routeTree.gen.ts` automatically.

Do NOT put page-level UI components here; those belong in `src/features/<feature>/`.
